# Ayo Sandbox & Agent Runtime Implementation Plan

## Status: Planning Complete - Ready for Implementation

## Executive Summary

This plan covers a major architectural evolution of ayo:
- **Sandbox Execution**: Agents run in isolated containers (Apple Container on macOS)
- **Zettelkasten Memory**: File-based memory with TOML frontmatter, SQLite index
- **Daemon Architecture**: Host daemon manages LLM inference, sandbox lifecycle
- **Provider System**: Pluggable backends for memory, sandbox, embedding, observers
- **Session Files**: JSONL-based session storage replacing SQLite

## Design Context (from discussion)

### Core Architecture
```
                            Host (User Session)
  ┌─────────────────────────────────────────────────────────────────┐
  │                      ayo daemon                                 │
  │  - LLM inference (local models, API keys)                       │
  │  - Session/memory persistence                                   │
  │  - Container lifecycle management                               │
  │  - Tool result routing                                          │
  └──────────────────────┬──────────────────────────────────────────┘
                         │ gRPC/Unix socket
  ┌──────────────────────┴──────────────────────────────────────────┐
  │                    Sandbox Pool (shared by default)             │
  │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐              │
  │  │  @ayo       │  │  @research  │  │  @custom    │              │
  │  │  - bash     │  │  - bash     │  │  - bash     │              │
  │  │  - tools    │  │  - tools    │  │  - tools    │              │
  │  └─────────────┘  └─────────────┘  └─────────────┘              │
  └─────────────────────────────────────────────────────────────────┘
```

### Key Decisions Made
- Base image: Busybox + POSIX tools + curl (lightweight, old-school)
- Installation: install.sh pulls busybox alongside other dependencies
- Languages: Per-agent, declared in config, installed on first use
- Network: Allowed by default
- Topology: One shared sandbox, break out agents as needed
- Lifecycle: ayo manages daemon transparently, sandbox stays warm
- Memory conflicts: Auto-merge, mark unclear for agent to ask naturally
- Session storage: JSONL files (not raw context dumps)
- Approval policies: Auto-accept everything (for now)

### Memory System Design
```
~/.local/share/ayo/memory/
├── index.md              # Auto-generated overview
├── topics/
│   └── go-preferences.md
├── facts/
│   └── api-key-location.md
├── corrections/
│   └── dont-use-cobra.md
└── .index.sqlite         # Embeddings + FTS (derived, rebuildable)
```

File format (TOML frontmatter):
```markdown
+++
id = "mem_01HX..."
created = 2024-01-15T10:30:00Z
topics = ["go", "testing"]
source = "session:abc123"
+++

User strongly prefers table-driven tests in Go.
```
