# Rough Idea

Create a set of contrived example projects that demonstrate all Ayo CLI features, then use these examples as the foundation for comprehensive documentation.

## Goals

1. **Example Projects**: Build working examples that exercise every Ayo feature
2. **Documentation Suite**: Author docs that use these examples as illustrations
3. **Quality Testing**: Examples double as integration tests for the build system
4. **User Education**: Examples progress from simple to complex

## Initial Thoughts

The examples should cover:
- Basic text-in/text-out agent (simplest case)
- Structured input with CLI flags
- Structured output generation
- File input/output handling
- Skills integration
- Hooks for lifecycle events
- Complex prompt templates
- Multiple input types (strings, integers, booleans, enums)
- Positional arguments

Documentation should include:
- Getting started / quickstart
- Project structure reference
- Configuration guide
- Schema definitions (input/output)
- CLI flag customization
- Skills documentation
- Hooks documentation
- Template functions reference
- Full API reference (generated code)
- Examples gallery with explanations
