# Update Manual Test Instructions

## Objective

Update the manual test instructions in the project root to reflect the Docker removal and native container approach.

## Context

The `MANUAL_TEST.md` file in the project root contains test instructions that likely reference Docker. These need to be updated to use Apple Container or the none provider.

## File to Modify

### MANUAL_TEST.md

**Review and update:**

1. **Sandbox test phases** - Replace Docker-specific tests with:
   - Apple Container tests (macOS 26+)
   - None provider tests (all platforms)
   - Provider auto-detection tests

2. **Prerequisites** - Update requirements:
   - Remove Docker installation requirement
   - Add Apple Container installation (optional, macOS 26+ only)
   - Document platform-specific requirements

3. **Test commands** - Replace any Docker CLI commands:
   - `docker run` → `container run`
   - `docker exec` → `container exec`
   - `docker ps` → `container list`
   - `docker stop` → `container stop`

4. **Expected outputs** - Update expected messages:
   - Provider selection output
   - Fallback behavior messages
   - Error messages for unavailable providers

## Example Updates

### Before (Docker-based)
```markdown
## Phase X: Sandbox Execution

Prerequisites:
- Docker installed and running

Steps:
1. Run `docker ps` to verify Docker is available
2. Start daemon: `ayo daemon start`
3. Check status shows Docker provider
```

### After (Native containers)
```markdown
## Phase X: Sandbox Execution

Prerequisites:
- macOS 26+ with Apple Container (optional, for isolated execution)
- OR any platform (will use host execution)

Steps:
1. Check container availability:
   - macOS: `container system status` (if Apple Container installed)
   - All platforms: Proceed without containers
2. Start daemon: `ayo daemon start`
3. Check status: `ayo status`
   - With Apple Container: Shows "apple" sandbox provider
   - Without: Shows "none" sandbox provider (host execution)
```
