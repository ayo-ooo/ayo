# Delete Docker Provider and All Docker References

## Objective

Completely remove the Docker sandbox provider and all Docker-related code from the codebase.

## Context

Docker is being replaced with native platform solutions. This ticket removes all Docker code to prevent confusion and ensure clean separation.

## Files to Delete

```
internal/sandbox/docker.go                    # Docker provider implementation
internal/sandbox/images/Dockerfile            # Docker image definition
```

## Files to Modify

### internal/sandbox/sandbox_test.go

Remove these test functions:
- `TestDockerProvider_Name`
- `TestDockerProvider_IsAvailable`
- `TestDockerProvider_Init_WhenNotAvailable`
- `TestDockerProvider_Create_WhenNotAvailable`
- `TestDockerProvider_Integration`
- `TestDockerProvider_Get_NotFound`
- `TestDockerProvider_AssignAgent`

Remove any imports of `dockerSandbox` or `DockerProvider` types.

### internal/sandbox/mounts/mounts.go

Remove:
- `DockerMountArgs()` function (lines 148-165 approximately)

### internal/sandbox/mounts/mounts_test.go

Remove:
- `TestDockerMountArgs` function and its test cases

### internal/sandbox/images/busybox.md

Remove all Docker-specific content:
- Docker build commands
- Docker pull commands
- Docker CLI operation descriptions
- References to Docker provider

Rewrite to focus on Apple Container image usage.

### install.sh

Remove:
- `setup_docker()` function entirely
- Any calls to `setup_docker`
- Docker availability checks

### internal/integration/harness.go

Remove:
- `WithDocker()` method
- Any Docker-specific test environment setup

### internal/integration/harness_test.go

Remove:
- `TestHarness_ExecDocker` test
- Any Docker-specific integration tests

### internal/providers/loader.go

Remove:
- Line 204: `configs["sandbox.apple-container"] = sandboxConfig` (keep this one)
- Remove any references to "docker" as a provider name

### AGENTS.md

Remove:
- Any mention of Docker as a sandbox provider
- Docker-specific configuration examples
- References to `docker.io` images (keep OCI image references)
