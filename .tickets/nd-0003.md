# Rewrite AppleProvider to Match Real Apple Container CLI

## Objective

Update `internal/sandbox/apple.go` to correctly use the Apple Container CLI based on the research documented in nd-0001.

## Context

The current AppleProvider implementation was written speculatively. After researching the actual Apple Container CLI from `.read-only/container/`, this implementation needs to be corrected.

## Current Implementation Issues (to verify)

Based on `.read-only/container/docs/command-reference.md`:

1. **Container creation**: Current code uses `container create --name NAME IMAGE` but may need different flags
2. **Starting containers**: May need `container start --attach` for interactive mode
3. **Volume mounts**: Uses `-v SRC:DST` but Apple Container uses `--volume` or `--mount`
4. **Network**: Uses `--network none` but Apple Container may use different syntax
5. **Exec**: Uses `container exec CONTAINER CMD` but may need different options
6. **Status checking**: Uses `container inspect` but output format may differ

## Reference: Apple Container CLI

From `.read-only/container/docs/command-reference.md`:

```bash
# Create and run a container
container run [options] <image> [<arguments> ...]

# Options for container run:
# -d, --detach        Run in background
# -v, --volume        Bind mount a volume
# --mount             Add a mount (format: type=<>,source=<>,target=<>)
# --network           Attach to network
# -c, --cpus          Number of CPUs
# -m, --memory        Memory limit

# Execute in running container
container exec [options] <container-id> <arguments> ...

# Options for exec:
# -w, --workdir       Set working directory
# -e, --env           Set environment variables
# -i, --interactive   Keep stdin open
```
