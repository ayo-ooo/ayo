# Research Apple Container CLI and Document API

## Objective

Thoroughly research the Apple Container CLI from `.read-only/container/` to understand:
1. Exact CLI commands and their arguments
2. How containers are created, started, stopped, and deleted
3. How exec works within running containers
4. How mounts (virtiofs) are configured
5. Network configuration options
6. Resource limits (CPU, memory)
7. Image management (pull, list, etc.)

## Context

The current `internal/sandbox/apple.go` implementation makes assumptions about the Apple Container CLI that may not match reality. The CLI is documented in `.read-only/container/docs/command-reference.md`.

## Prerequisites

- Clone apple/container to .read-only/container/ (DONE)
- macOS 26+ on Apple Silicon for testing (optional for research)

## Tasks

1. **Read command-reference.md thoroughly**
   - File: `.read-only/container/docs/command-reference.md`
   - Focus on: container run, create, start, stop, delete, exec, list, inspect

2. **Analyze CLI source code**
   - Location: `.read-only/container/Sources/ContainerCommands/`
   - Key files:
     - `Container/ContainerCreate.swift`
     - `Container/ContainerExec.swift`
     - `Container/ContainerRun.swift`
     - `Container/ContainerStart.swift`
     - `Container/ContainerStop.swift`
     - `Container/ContainerDelete.swift`

3. **Document differences from current implementation**
   - Current apple.go uses `container create --name NAME --volume SRC:DST ...`
   - Verify this matches actual CLI
   - Check if we need `container run` vs `container create` + `container start`

4. **Create API documentation**
   - Create `internal/sandbox/APPLE_CONTAINER.md` documenting:
     - Required macOS version (26+)
     - Required architecture (arm64)
     - CLI commands used by ayo
     - Expected output formats
     - Error handling patterns

## Expected Output

A markdown document (`internal/sandbox/APPLE_CONTAINER.md`) that serves as the authoritative reference for implementing AppleProvider.

## Definition of Done

- [ ] command-reference.md fully reviewed
- [ ] Key Swift source files analyzed
- [ ] APPLE_CONTAINER.md created with complete CLI documentation
- [ ] Gaps identified between current apple.go and real CLI
