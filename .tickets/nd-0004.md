# Research Linux Native Container Solutions (LXC/LXD)

## Objective

Research and document Linux native container solutions as alternatives to Docker for the Linux platform.

## Context

While macOS uses Apple Container, Linux needs its own native solution. Options include:
- **LXC** (Linux Containers) - Low-level container runtime
- **LXD** - Higher-level LXC management daemon
- **systemd-nspawn** - systemd's container solution
- **crun/runc** - OCI runtime (used by Podman)
- **Podman** - Docker alternative (considered too heavy?)

## Research Questions

1. **LXC vs LXD**
   - What's the minimum installation requirement?
   - Is LXC available on all major distros (Ubuntu, Fedora, Arch, Debian)?
   - What's the CLI interface?
   - How do we detect availability?

2. **systemd-nspawn**
   - Available on systemd-based distros (most modern Linux)
   - Simpler than LXC?
   - Good for lightweight containers?

3. **Distribution considerations**
   - Ubuntu/Debian: apt packages
   - Fedora/RHEL: dnf packages
   - Arch: pacman packages
   - Alpine: apk packages (no systemd)

4. **Permission requirements**
   - Do we need root?
   - Can we use rootless containers?
   - User namespace considerations?

## Candidate Solutions

### Option A: LXC (Preferred for low-level)

```bash
# Ubuntu/Debian
apt install lxc lxc-utils

# Fedora
dnf install lxc lxc-extra

# CLI
lxc-create -t download -n mycontainer -- -d ubuntu -r focal -a amd64
lxc-start -n mycontainer
lxc-attach -n mycontainer -- /bin/bash
lxc-stop -n mycontainer
lxc-destroy -n mycontainer
```

Pros:
- Very low-level, minimal overhead
- Available on all major distros
- Well-documented

Cons:
- Requires root or complex unprivileged setup
- Image management is manual

### Option B: LXD (Higher-level LXC)

```bash
# Ubuntu
snap install lxd

# CLI
lxc launch ubuntu:22.04 mycontainer
lxc exec mycontainer -- /bin/bash
lxc stop mycontainer
lxc delete mycontainer
```

Pros:
- Easier to use than raw LXC
- Built-in image management
- Can run unprivileged

Cons:
- Snap-based on Ubuntu (may not be available everywhere)
- Heavier than raw LXC

### Option C: systemd-nspawn

```bash
# Create container from tarball
machinectl pull-tar URL mycontainer

# Run
systemd-nspawn -D /var/lib/machines/mycontainer /bin/bash

# Or as service
machinectl start mycontainer
machinectl shell mycontainer
machinectl stop mycontainer
```

Pros:
- Built into systemd (no extra install on most distros)
- Very lightweight
- Good integration with systemd services

Cons:
- Not available on non-systemd distros (Alpine, Void, etc.)
- Less flexible than LXC
