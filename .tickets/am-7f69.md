# Add squad parameter to flow agent steps

Flow YAML/bash scripts should support specifying a squad context for agent steps. This allows flows to dispatch work to squads as part of a pipeline.

## Status: ALREADY IMPLEMENTED

Research shows this feature is already complete:

### Existing Implementation

1. **FlowStep.Squad field** - internal/flows/spec.go:75-79
   ```go
   Squad string `yaml:"squad,omitempty" json:"squad,omitempty"`
   ```

2. **AgentInvoker.InvokeInSquad()** - internal/flows/yaml_executor.go:22-23

3. **Squad execution branch** - internal/flows/yaml_executor.go:345-348:
   ```go
   if step.Squad != "" {
       response, invokeErr = e.AgentInvoker.InvokeInSquad(ctx, step.Squad, step.Agent, prompt)
   } else {
       response, invokeErr = e.AgentInvoker.Invoke(ctx, step.Agent, prompt)
   }
   ```

4. **SandboxAwareInvoker.InvokeInSquad()** - internal/daemon/invoker.go:55-81

5. **Parse test with squad** - internal/flows/parse_test.go:210-257

6. **Previous ticket am-mh6x** - closed, squad field already added

## Example Usage

```yaml
steps:
  - id: build
    type: agent
    agent: "@developer"
    squad: "#frontend"
    prompt: "Build the login component"
```
