# Add squad parameter to flow agent steps

Flow YAML/bash scripts should support specifying a squad context for agent steps. This allows flows to dispatch work to squads as part of a pipeline.

## Design

In flow step parsing, add optional 'squad' field for agent type steps. When executing, if squad is specified, route through squad dispatch instead of direct agent invocation. Example: '# squad: dev-team' comment in bash flow or 'squad: "#dev-team"' in YAML.

## Acceptance Criteria

- Flow agent steps can include squad: field
- Agent runs in squad sandbox context when squad specified
- Flow receives output from squad dispatch
